2026 Data Migration — Step by Step
Step 1. Export Client Register tab from master Excel as CSV
Step 2. Upload below → preview → import clients
Step 3. Export Sessions Log tab from master Excel as CSV
Step 4. Upload below → preview → import sessions
Step 5. Add group cases manually via Add/Edit Case tab (only 6 — faster than CSV)
⚠️ Do NOT include FIN numbers, contact numbers, or employer details in any export. These stay in the master Excel only.
Phase 1 — Import Client Register
Export the Client Register tab from the master Excel as CSV. The tool will automatically map columns and convert Excel dates. Expected columns: Case ID, Status, Date Opened, Date Closed, Counsellor, Name, Age, Nationality, Language, Pass Type, Location / Dorm, Referral Source, Primary Issue, Overall Risk Level, Reason for Closure, Feedback Collected
Phase 2 — Import Sessions Log
Export the Sessions Log tab from the master Excel as CSV. Session Notes will be imported as the S (Subjective) field — no SOAP structure expected for historical sessions. Expected columns: Session ID, Case ID, Client Name, Counsellor, Date, Contact #, Mode, Duration (min), Distress Start, Distress End, Session Notes, Outcome, Follow-up Date
Close a Case
Closing a case marks it as Closed, hides it from counsellor dropdowns, and starts the 5-year retention clock. Only close cases where counselling has genuinely ended. This action can be reversed by Durga if needed.
